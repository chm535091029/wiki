# add-new-knowledge

当用户提供一篇研究论文（PDF 或文本）或技术文章，要求阅读分析并整理为结构化的知识笔记时，使用此 skill。

## 前置条件

- 如果是 PDF 文件，先使用 `extract-pdf` skill 提取文本内容（调用 `extract_pdf_text.py` 脚本）
- 如果是文本内容（用户直接粘贴或已有文本文件），直接使用 `read_file` 读取

## 步骤

### 1. 完整阅读全文

- 使用 `read_file` 读取提取的文本文件或用户提供的内容，确保完整阅读所有页面/段落
- 特别关注：摘要（Abstract）、引言（Introduction）、方法/实验设计（Methodology/Experiments）、结果（Results）、讨论（Discussion）、结论（Conclusion）
- 对于长文件，分段阅读，确保不遗漏关键信息

### 2. 分析技术背景（Background）

从论文中提取以下信息：

- **研究背景**：该领域当前的技术状态是什么？已有方法有哪些？
- **核心问题**：作者指出的研究空白（research gap）或尚未解决的关键问题是什么？
- **相关工作**：引用了哪些重要工作？它们各自有什么局限？
- **研究动机**：为什么这个问题重要？为什么现在解决它？

### 3. 分析核心技术内容（Technical Content）

提取并归纳：

- **整体设计/架构**：系统的核心设计理念、架构图式（如模块化设计、多 Agent 框架等）
- **核心算法/方法**：关键的技术创新点，用表格或分点清晰对比
- **实验设置**：使用的数据集、评估指标、基线方法、实现细节等
- **关键结果**：用表格或分点列出主要实验结果，附上具体数值
- **消融/分析**：如果论文做了消融实验或深入分析，提取关键结论
- **公式/算法/伪代码**（如有）：提取关键公式和算法流程

### 4. 提取"特别标签观点"（Key Insights / Unique Viewpoints）

这是最重要的步骤。从论文中筛选出**最有特色、反常识、或具有深刻洞见的观点**，每一条应包含：

- **标签**：为该观点定义一个或多个简洁的标签（如 `Grep-Advantage`, `Harness-Retrieval-Interaction`）
- **核心论点**：用一句话概括该观点的核心主张
- **关键证据**：论文中支持该观点的具体数据或实验发现
- **深层含义**：这个观点意味着什么？对实践者有什么启示？

典型筛选标准：
- ✅ 与主流认知相反的观点（如"grep 比 vector 检索更适合 agent"）
- ✅ 揭示了系统设计中容易被忽略的交互效应（如"harness 不是被动基础设施"）
- ✅ 提出了新的分析视角或框架性认知（如"程序化交付改变任务本质"）
- ✅ 对现有建议/实践提出了条件化限制（如"默认使用向量检索需要条件化"）
- ✅ 具有跨领域适用性的普适性洞见

### 5. 格式化为知识文件

按照以下 YAML frontmatter + Markdown 结构写入知识库文件：

```markdown
---
aliases: [别名1, 别名2, 中文别名]
tags: [标签1, 标签2, 标签3]
related:
  - "./相关文件1.md"
  - "../目录/相关文件2.md"
---

# 论文/文章标题

## 论文基本信息

- **论文标题**: ...
- **作者**: ...
- **发表**: ...

---

## 一、技术背景

...

## 二、技术内容详解

...

## 三、特别标签观点与核心关联内容

### 观点 1：观点标题

- **标签**: `Tag1`, `Tag2`
- **核心论点**: ...
- **关键证据**: ...
- **深层含义**: ...

### 观点 2：观点标题

...

## 四、总结与局限

...

## 五、关键引用

...

## 六、对实践的启示（可选）

...
```

### 6. 文件存放路径规则

- 将文件保存在 `knowledge/` 目录下的适当子目录中
- 根据内容主题选择子目录：
  - `knowledge/03-research/Agent/` — Agent 相关
  - `knowledge/03-research/Foundation/` — 模型基础
  - `knowledge/03-research/RAG/` — RAG/检索相关
  - `knowledge/02-deployment/` — 部署相关
  - `knowledge/01-training/` — 训练相关
- 命名使用英文小写 + 连字符，如 `agentic-search-harness.md`
- 在 `aliases` 中包含中英文别名方便检索
- 在 `related` 中关联已有相关文件

### 7. 清理临时文件

- 如果使用了 PDF 提取的临时文本文件，用 `execute_command` 删除（如 `del output\xxx.txt`）

## 质量检查清单

- [ ] 是否完整阅读了全文，没有遗漏关键章节
- [ ] 技术背景是否清晰涵盖了现状、空白、动机
- [ ] 技术内容是否覆盖了架构、方法、实验、结果
- [ ] 是否提取了 5-10 个有深度的特别标签观点
- [ ] 每个观点是否都有具体的证据支持（数值/实验/引用）
- [ ] YAML frontmatter 的 aliases 和 tags 是否合理且完整
- [ ] related 是否关联了相关文件
- [ ] 文件路径是否正确
- [ ] 临时文件是否已清理
