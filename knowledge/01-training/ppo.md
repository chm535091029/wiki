# PPO (Proximal Policy Optimization)

**核心思想**：通过**裁剪机制（Clipping）**限制策略更新的幅度，避免因更新步过大导致性能崩溃。

PPO是目前LLM对齐阶段应用最广泛的强化学习算法之一。

## 算法原理

PPO采用了一种"保守"的策略更新方式：

```
L^{CLIP}(\theta) = E[min(r_t(θ) * A_t, clip(r_t(θ), 1-ε, 1+ε) * A_t)]
```

其中 `clip()` 函数将策略比率限制在 `[1-ε, 1+ε]` 范围内，防止单次更新过大。

## 代码生成任务下的数据要求

| 要求 | 说明 |
|------|------|
| prompt-response对 | 需要构建高质量的训练对 |
| 代码逻辑清晰 | response需要包含明确的代码逻辑和注释 |
| 场景覆盖 | 偏好数据应覆盖多种编程语言和场景 |

## 特点

- **训练范式**：传统RL（reward + critic）
- **数据需求**：SFT模型 + reward数据
- **训练复杂度**：高（多阶段）
- **适用场景**：复杂任务对齐
