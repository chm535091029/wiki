# DeepSeek V3.2

> **论文链接**: [arXiv:2512.02556](https://arxiv.org/pdf/2512.02556)  
> **核心关键词**: DeepSeek Sparse Attention (DSA), 强化学习 (RL), Agentic Task Synthesis, 推理能力

DeepSeek V3.2 通过三大技术突破，实现了与 GPT-5 相当的性能，并在特定推理任务上达到了 Gemini-3.0-Pro 的水平。

---

## 一、技术背景

在 DeepSeek-V3.2 发布之前，大语言模型（LLM）领域正面临三个核心挑战，尤其是开源模型：

| 挑战 | 描述 |
|------|------|
| **架构效率瓶颈** | 传统的 Vanilla Attention（全注意力机制）在处理长文本（Long-context）时计算复杂度过高，严重限制了推理效率和长序列的后训练（Post-training） |
| **后训练投入不足** | 相比于闭源模型（如 GPT-5 等），开源模型在强化学习阶段的计算资源投入比例较低，难以激发出极限的逻辑推理能力 |
| **Agent 任务数据稀缺** | 将高阶推理能力（Reasoning）转化为实际的工具使用（Tool-use）和智能体操作（Agentic tasks）时，缺乏大规模、高质量且可自动验证的合成数据 |

## 二、技术核心内容

### 1. DSA (DeepSeek Sparse Attention) —— 架构创新

DSA 是 DeepSeek V3.2 引入的核心架构创新，通过结构化稀疏设计大幅提升长文本处理效率。

#### 1.1 分层注意力策略（Hierarchical Strategy）

DSA 采用"混合布局"来平衡全局视野和计算效率：

| 层类型 | 位置 | 机制 | 作用 |
|--------|------|------|------|
| **滑动窗口层** | 模型底层（靠近输入） | 窗口大小 4096，Token $i$ 只看 $[i-4096, i]$ 范围 | 处理基础语法和近邻逻辑 |
| **稀疏全局层** | 模型中高层 | 选取特定索引（如每隔 $k$ 个 Token 选一个）作为"基准站" | 跨远距离传输信息 |

#### 1.2 块稀疏计算（Block-Sparse）

为避免 GPU 内存访问不连续的问题，DSA 采用块化设计：

```
计算单元：将 L×L 的注意力矩阵切分成 64×64 或 128×128 的小块
掩码决策：模型先预测哪些 Block 包含关键信息
计算跳过：对不重要的 Block，GPU 直接跳过加载和计算
```

#### 1.3 动态负载均衡（Dynamic Load Balancing）

DeepSeek 的"独门秘籍"：在分布式训练中，稀疏性会导致不同显卡计算负载不均。

- **问题**：某个显卡计算量过大时，整个集群会停下来等它（木桶效应）
- **解法**：引入动态路由机制，过载的计算块实时分配到相对空闲的计算节点
- **效果**：在 128K 长度训练时，集群硬件利用率依然维持在 **90% 以上**

#### 1.4 与 FlashAttention 的对比

| 方案             | 原理              | 效果               |
| -------------- | --------------- | ---------------- |
| FlashAttention | 依然计算全矩阵，只是计算得很快 | 优化 IO 读写速度       |
| DSA            | 直接减少计算总量        | 砍掉 80%-90% 的无效计算 |
| **Flash-DSA**  | 结合两者优势          | 既优化读写，又减少计算      |

#### 1.5 核心原理

DSA 将长文本复杂度从 $O(L^2)$ 降到了 $O(L \times \text{log}(L))$ 或接近 $O(L)$：

> 想象你在读一本 500 页的书：
> - **全注意力**：每读到一个字，都要回头翻一遍前面所有的字
> - **DSA**：主要看这一页的内容（局部），同时记住之前每章的大纲和关键结论（全局跳跃点），自动跳过没意义的广告页（块稀疏）

### 2. 规模化强化学习框架 (Scalable RL Framework)

DeepSeek-V3.2 特别强调了后训练阶段的计算规模。

#### 2.1 核心数据

| 指标 | 数值 |
|------|------|

### 3. 大规模智能体任务合成流水线 (Agentic Task Synthesis Pipeline)

为增强模型在现实交互环境中的表现，团队开发了一套自动化合成系统。

#### 3.1 技术路线

| 阶段 | 说明 |
|------|------|
| **任务设计** | 构建超过 1,800 个模拟环境，覆盖多场景 agent 任务 |
| **轨迹合成** | 生成约 85,000 个复杂的 Prompt 及多样化的执行轨迹 |
| **质量筛选** | 所有代码和操作步骤必须通过严格的 Regression tests |

#### 3.2 核心组件

| 组件 | 说明 |
|------|------|
| **Lightning Indexer** | 高效的索引机制 |
| **Fine-grained Token Selection** | 细粒度 Token 选择，减少无效计算 |

#### 3.3 设计理念

- **动态稀疏**：根据输入动态激活计算路径
- **细粒度选择**：Token 级别精细化处理
- **效率提升**：减少无意义的计算开销

#### 3.4 效果

显著提升了模型在以下方面的能力：
- 工具调用（Tool-use）
- 指令遵循（Instruction-following）
- 复杂交互环境中的鲁棒性

## 三、训练流程

```
┌─────────────────────────────────────────────────────────────┐
│                    持续预训练 (Continued Pre-training)       │
├─────────────────────────────────────────────────────────────┤
│  阶段一：2.1B Token 密集型 (Dense) 预热                       │
│  阶段二：943.7B Token 稀疏型 (Sparse) 训练                    │
│          → 使模型适应 DSA 架构并对齐长上下文数据              │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                 专家蒸馏 (Specialist Distillation)           │
│  → 利用数学、代码等领域专家模型生成高质量 SFT 数据            │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                 混合强化学习 (Mixed RL)                       │
├─────────────────────────────────────────────────────────────┤
│  可验证任务（数学/编程）→ 基于规则的奖励 (Rule-based)        │
│  通用任务          → 基于准则的生成式奖励模型 (Rubric-driven) │
└─────────────────────────────────────────────────────────────┘
```
