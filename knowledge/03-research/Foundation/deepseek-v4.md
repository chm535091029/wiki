# DeepSeek-V4 技术框架

论文解读：https://zhuanlan.zhihu.com/p/2030982954617414764

## 动机背景

### 问题与挑战
1. **Test-time Scaling瓶颈**：推理模型的出现建立了test-time scaling新范式，但标准注意力机制的二次复杂度成为超长上下文的主要瓶颈
2. **超长上下文需求**：复杂Agent工作流、跨文档分析等长时程任务需要高效支持超长上下文
3. **现有开源模型的局限**：虽然开源模型在通用能力上取得进展，但在处理超长序列时的架构效率问题仍是关键障碍

### 设计目标
- 突破超长上下文的效率壁垒
- 高效支持100万token上下文
- 为test-time scaling和长时程任务探索奠定基础

## 核心技术框架

### 1. 整体架构

```
输入Token → Embedding → CSA/HCA混合注意力 → DeepSeekMoE → MTP模块 → 预测头
                    ↑                              ↑
              mHC增强残差连接                    LM Loss
```

DeepSeek-V4保留了Transformer架构和Multi-Token Prediction模块，同时引入三项关键升级：
- Manifold-Constrained Hyper-Connections (mHC)
- 混合注意力（CSA + HCA）
- Muon优化器

### 2. Mixture-of-Experts (MoE)

采用DeepSeekMoE范式：
- 细粒度路由专家 + 共享专家
- 激活函数从 Sigmoid 改为 Sqrt(Softplus)
- 无辅助loss的负载均衡策略 + 序列级平衡loss
- 早期Transformer层使用Hash路由的MoE层

### 3. Multi-Token Prediction (MTP)

与DeepSeek-V3保持一致，通过多token预测增强训练信号。

## 核心技术创新

### 1. Manifold-Constrained Hyper-Connections (mHC)

**问题**：标准HC在堆叠多层时常出现数值不稳定

**解决方案**：将残差映射矩阵约束到双随机矩阵流形（Birkhoff多面体）

```
B^l ∈ M ≔ {M ∈ R^(n×n) | M1_n = 1_n, 1_n^T M = 1_n^T, M ≥ 0}
```

**关键特性**：
- 谱范数有界（≤1），非扩展变换增强数值稳定性
- 流形在乘法下闭合，保证深层堆叠的稳定性
- 输入/输出变换通过Sigmoid约束为非负有界
- 动态参数化：静态偏置 + 动态分量

**参数约束**：通过Sinkhorn-Knopp算法将矩阵投影到双随机矩阵流形（迭代20次）

### 2. 混合注意力机制

#### Compressed Sparse Attention (CSA)
1. **KV压缩**：每m个token的KV缓存压缩为1个条目
2. **稀疏选择**：通过Lightning Indexer选择top-k压缩KV条目
3. **DeepSeek Sparse Attention (DSA)**：进一步加速

架构组件：
- 压缩索引器键
- 多查询注意力
- 分组输出投影
- 滑动窗口KV条目（增强局部依赖）

#### Heavily Compressed Attention (HCA)

**设计动机**：HCA追求极端压缩，通过将每 m'（>> m）个 token 的 KV 缓存合并为单个条目，在超长上下文场景下实现极致效率。

**核心流程说明**：

| 步骤 | 组件 | 功能 |
|------|------|------|
| 1 | Token级压缩器 | 将m'个token的KV合并为1个条目 |
| 2 | Shared KV MQA | 低秩压缩→展开为多头→执行MQA |
| 3 | 滑动窗口KV | 补充最近n_win个未压缩token |
| 4 | 分组输出投影 | 将c·n_h维输出压缩到d维 |

**1. 压缩键值条目**

HCA的压缩策略与CSA类似，但采用更大的压缩率 m'（远大于CSA的m），且不执行重叠压缩。

给定输入隐藏状态 H ∈ R^(n×d)：

- 计算原始KV条目 C ∈ R^(n×c) 和压缩权重 Z ∈ R^(n×c)：
  ```
  C = H · W_KV
  Z = H · W_Z
  ```

- 每 m' 个 KV 条目根据压缩权重和可学习位置偏置 B ∈ R^(m'×c) 压缩为一个，产生 C_Comp ∈ R^(n/m' × c)：
  ```
  S = Softmax_row(Z[m'·i : m'·(i+1)-1] + B)
  C_Comp[i] = Σ(S[j] ⊙ C[j])  for j = m'·i to m'·(i+1)-1
  ```

通过压缩操作，HCA将序列长度压缩到 **1/m'** 倍。

**2. 共享KV多查询注意力 (Shared KV MQA)**

HCA使用与CSA相同的Shared KV MQA策略：

- 对查询token t，以低秩方式生成注意力查询：
  ```
  c_Q = h_t · W_DQ          # 压缩潜在向量
  q_t = c_Q · W_UQ          # 展开为多头查询
  ```

- 执行MQA，压缩KV条目同时作为key和value：
  ```
  o_t,i = CoreAttn(query=q_t,i, key=C_Comp, value=C_Comp)
  ```

**3. 分组输出投影 (Grouped Output Projection)**

由于 c·n_h 非常大，直接投影到 d 维会带来巨大计算负担。HCA采用分组策略：

1. 将 n_h 个输出分成 g 个组
2. 每组输出 o_G ∈ R^(c·n_h/g) 投影到 d_g 维中间输出（d_g < c·n_h/g）
3. 最后将中间输出投影到最终注意力输出 ŏ_t ∈ R^d

**4. 辅助技术细节**

- **Query/KV归一化**：核心注意力操作前对查询和压缩KV的每个head执行RMSNorm，避免注意力logits爆炸
- **部分RoPE**：仅对查询/KV向量的最后64维应用旋转位置编码
- **滑动窗口注意力**：为每个查询token额外生成 n_win 个未压缩的KV条目（对应最近n_win个token），与压缩KV条目拼接使用，增强局部细粒度依赖建模
- **Attention Sink**：添加可学习sink logits到注意力分母，允许各查询头调整总注意力分数不为1甚至接近0

**5. 效率优势**

| 指标 | 标准注意力 | HCA |
|------|-----------|-----|
| KV Cache 大小 | O(n·c) | O(n/m'·c) |
| 注意力计算复杂度 | O(n²·c) | O(n·(n/m')·c) |
| 序列压缩比 | 1× | 1/m' × |

在100万token上下文中，HCA配合混合存储格式（RoPE维度BF16 + 其余FP8），可将KV缓存大小减少到基准的约 **2%**。

#### 效率优化
- **混合存储格式**：RoPE维度用BF16，其余用FP8，减少近一半KV缓存
- **索引器FP4精度**：极端长上下文中加速
- **注意力sink**：可学习sink logits避免注意力分数爆炸

### 3. Muon优化器

**核心算法**：
```
G_t = ∇_W L_t(W_{t-1})
M_t = μM_{t-1} + G_t
O'_t = HybridNewtonSchulz(μM_t + G_t)  # Nesterov技巧
O_t = O'_t · √(max(n,m) · γ)           # 更新RMS重缩放
W_t = W_{t-1} · (1 - ηλ) - ηO_t        # 权重衰减和更新
```

**Hybrid Newton-Schulz迭代**：
- 阶段1（前8步）：系数(3.4445, -4.7750, 2.0315)快速收敛
- 阶段2（后2步）：系数(2, -1.5, 0.5)稳定奇异值到1

**配置**：
- AdamW：embedding、预测头、mHC静态偏置、RMSNorm权重
- Muon：其他所有模块

## 训练基础设施

### 1. 专家并行通信优化
- **细粒度EP方案**：将专家分组为wave，通信与计算完全流水线化
- **理论加速**：1.92×（vs Comet的1.42×）
- 延迟可被计算隐藏的条件：C/B ≤ V_comp/V_comm

### 2. TileLang内核开发
- DSL平衡开发效率与运行时效率
- Host Codegen减少调用开销（验证开销从微秒级降至<1微秒）
- SMT求解器辅助形式化整数分析
- 位级可复现性

### 3. 高性能确定性内核
- **Batch Invariance**：token输出位置无关的位级一致性
- **Determinism**：训练过程完全确定性
- 双kernel策略解决attention的wave量化问题

### 4. 训练框架创新
- **Muon ZeRO混合策略**：密集参数用knapsack分配，MoE参数独立优化
- **mHC优化**：融合内核 + 选择性重计算 + DualPipe调整，墙钟时间开销仅6.7%
- **上下文并行两阶段通信**：解决压缩注意力跨rank边界问题
- **张量级激活检查点**：自动微分框架支持

### 5. 推理框架
- **异构KV缓存结构**：压缩KV + 滑动窗口KV + 状态缓存
- **磁盘KV缓存**：支持超长上下文
- **共享前缀复用**：高效处理多轮对话
