# DeepSeek V4 (Engram)

DeepSeek V4引入Engram架构，旨在解决计算与记忆的耦合问题。

## 背景：计算与记忆的耦合

### 核心矛盾

| 能力类型 | 特点 | 问题 |
|----------|------|------|
| **推理/计算** | 需要深层矩阵运算 | 计算密集 |
| **事实性记忆** | 静态检索式 | 效率低下 |

### 现状问题

- 传统FFN承载所有记忆和计算
- 存储容量受限于模型参数
- 事实检索效率低

## Engram解决方案

### 解决的问题

| 问题 | Engram方案 |
|------|------------|
| **存储瓶颈** | 嵌入表扩展至万亿级参数 |
| **推理成本** | 替代部分FFN工作负载 |
| **显存优化** | 记忆库放在DDR而非HBM |

### 核心思想

打破传统Transformer的单一计算范式，将**计算**与**记忆**分离。

## 方法框架

| 技术 | 说明 |
|------|------|
| **N-gram索引机制** | O(1)复杂度查找 |
| **大规模嵌入表** | 稀疏激活，减少计算量 |
| **条件预测门控** | 判断是否需要查表 |
| **U-型扩展律** | 计算与记忆的最优比例 |

### N-gram索引

```
传统查找：O(n) - 逐个扫描
Engram查找：O(1) - 哈希索引
```

### 稀疏激活

```
输入 → 门控机制 → 激活少量嵌入 → 输出
        ↓
    大部分计算跳过
```

## 与Transformer骨干的集成

### 架构

```
┌─────────────┐
│   Input     │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  Transformer │ ← 推理/计算
│    Backbone  │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│    Engram   │ ← 记忆存储
└──────┬──────┘
       │
       ▼
┌─────────────┐
│   Output    │
└─────────────┘
```

### 核心要点

- **Engram与FFN并行工作**
- 共同构成模型的计算核心
- 记忆库独立于Transformer参数

## 技术意义

1. **存储扩展**：突破模型参数限制
2. **效率提升**：O(1)检索速度
3. **成本降低**：DDR存储成本远低于HBM
