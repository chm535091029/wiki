这篇文章全面解析了Anthropic Claude API的Prompt Caching（提示缓存）机制，这是一种旨在通过复用已处理的提示内容来大幅降低AI服务使用成本和延迟的核心技术。

**一、核心目标与价值**
Prompt Caching旨在解决大语言模型（LLM）API调用中的核心矛盾：每次调用都需要重新计算整个提示，即使其中绝大部分内容（如系统提示、工具定义、项目文档、对话历史）与前次请求相同。这导致严重的计算浪费、高延迟和高成本。该机制通过缓存已处理的提示前缀，在后续请求中直接复用，实现三大核心收益：
1.  **成本大幅降低**：缓存命中时，输入token费用可降低高达90%（仅需支付原价的10%）。
2.  **延迟显著减少**：首token延迟（TTFT）可降低高达79%。
3.  **吞吐量有效提升**：缓存命中的请求不计入API速率限制配额。

**二、核心机制**
1.  **基本原理**：对提示中标记为可缓存的部分计算哈希值并存储其计算的中间状态。当新请求的前缀哈希与缓存匹配时，直接从缓存读取状态，跳过重复计算。
2.  **两种使用模式**：
    *   **自动缓存**：在请求体顶层添加`cache_control`字段，系统自动管理缓存断点，是多轮对话的推荐方案。
    *   **显式缓存断点**：在具体的内容块上标记`cache_control`，可精细控制多个独立部分的缓存。
3.  **关键约束**：要求**精确匹配**（100%相同的文本/图片），不影响输出，遵循零数据留存政策，并进行了工作区级隔离。

**三、工程架构深度解析**
1.  **前缀哈希匹配**：
    *   缓存只在断点处写入。
    *   缓存读取时，若当前断点未命中，会**向后回溯**检查最多20个历史内容块位置。
2.  **层级与失效**：缓存遵循`工具定义(Tools) → 系统消息(System) → 对话消息(Messages)`的层级顺序，上层变更会导致其**下所有缓存失效**。最佳实践是将最稳定的内容（如工具定义）放在最前。
3.  **回溯窗口限制**：每个断点最多回溯检查20个内容块。在长对话中，若内容增长过快可能导致早期缓存超出窗口而丢失，需**手动添加额外断点**来维持命中（每个请求最多支持4个断点）。

**四、缓存管理（TTL与计费）**
1.  **两种TTL**：
    *   **5分钟缓存（默认）**：写入成本为基准输入价格的1.25倍，适用于高频交互。
    *   **1小时缓存（扩展）**：写入成本为基准价格的2倍，适用于低频或长间隔任务。
2.  **计费模型**：采用分层计费，`缓存读取`享受0.1倍（即一折）的超低费率。响应中的`usage`字段会详细展示`cache_read_input_tokens`（读取量）、`cache_creation_input_tokens`（写入量）和`input_tokens`（新处理量），便于监控。
3.  **最小可缓存长度**：不同模型有最小token数要求（1024至4096不等），低于此值会静默跳过缓存。

**五、在Claude Code中的集成与应用**
Claude Code是Prompt Caching的典型应用。其架构中稳定不变的部分（如`CLAUDE.md`项目文档、系统提示词、工具定义）会自动成为缓存目标，在多轮编码对话中被反复命中，从而显著降低单次交互的边际成本。文章提供了企业级使用Claude Code的成本数据（月均$150-250/开发者）和优化策略（如使用`/clear`命令、管理子Agent、按需加载工具）。

**六、性能与成本效益**
官方基准测试显示，在以下场景中收益显著：
*   **长文档问答（100K token）**：成本降低90%，TTFT从11.5秒降至2.4秒。
*   **Few-shot提示（10K token示例）**：成本降低86%，TTFT降低31%。
*   **多轮对话（10轮）**：成本降低53%，TTFT降低75%。
成本模型分析表明，在典型的Claude Code使用场景中，采用缓存可节省超过50%的成本。

**七、最佳实践与产品场景**
1.  **最佳实践**：
    *   **结构优化**：遵循“稳定内容在前，变化内容在后”的原则组织提示。
    *   **规避陷阱**：避免在缓存前缀中包含动态内容（如时间戳）；确保JSON序列化稳定；监控缓存命中率（>80%为优）；注意并行请求的缓存竞争和回溯窗口限制。
2.  **典型应用场景**：长文档问答系统、Few-shot提示工程、RAG（检索增强生成）管线、客户支持Agent、自动化代码审查以及AI编程助手（如Claude Code）。

**八、未来展望**
技术演进可能向**语义级缓存**、**更长TTL**、**更大回溯窗口**、**跨会话缓存**和**自动断点优化**方向发展。Prompt Caching正推动AI服务计费模式从“按次计费”向“按增量计费”转变，使得长上下文窗口和复杂Agent架构在经济上更为可行。

总而言之，Prompt Caching是一项通过**计算状态复用**来优化推理效率的底层基础设施技术，是降低大语言模型应用运营成本、提升响应速度的关键武器。
